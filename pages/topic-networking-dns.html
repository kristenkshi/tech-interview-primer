DNS

DNS (Domain Name System) is a distributed hierarchical system that translates human-readable domain names, such as www.example.com, into machine-readable IP addresses, like 192.0.2.1, enabling efficient communication between devices on computer networks. It acts as a phonebook of the internet, mapping domain names to their corresponding IP addresses.

What happens when you type a URL into a browser?

When a user enters a domain name into a web browser, the DNS resolution process begins. The browser sends a DNS query to the configured DNS resolver (typically provided by the internet service provider or a public DNS server). The resolver checks its cache for the corresponding IP address. If it doesn't have the information, it sends queries to other DNS servers in a recursive manner.

Suppose the resolver doesn't have the IP address for "www.example.com". It starts by querying the root DNS servers, which direct it to the top-level domain (TLD) servers responsible for ".com". The resolver then contacts the TLD servers to obtain the authoritative DNS server for "example.com". Finally, the resolver queries the authoritative DNS server to retrieve the IP address for "www.example.com".

Once the resolver receives the IP address, it caches the information for future use and returns it to the browser. The browser can then establish a connection to the web server at the IP address obtained through DNS resolution, allowing the user to access the desired website.

In this example, DNS resolution is used to translate the domain name "www.example.com" into its corresponding IP address, facilitating the connection between the user's browser and the web server hosting the example.com website.
